7. Risk Assessment & Recommendations

CRITICAL: Multiple media migration gaps identified

Action Items

Finding Severity Impact
3,886 EF images missing alt text CRITICAL SEO, accessibility, WCAG compliance
~1,158 oversized images on staging CRITICAL Page load performance, Core Web Vitals
Staging has 3.3x fewer media items than live HIGH Content loss on migration go-live

  1. Alt text remediation — 3,948 images across all sites lack alt text (3,886 of these are on Events Fantastic alone). The staging site's 99.7% coverage proves the process works — it needs to be applied retroactively to the live EF library, or ensured during migration.
  2. Image optimisation on staging — 1,158 oversized images on staging (25.6% of all staging media) must be compressed before go-live. Recommend WebP conversion with JPEG fallback. Target: all images under 300KB for standard images, under 500KB for hero/full-width images.
  3. Staging media sync — Run the comparison scripts (media_audit.js or media_audit.py) to identify the exact files missing from staging. The raw count gap (14,978 live vs 4,519 staging) suggests thousands of images need to be migrated. Priority should be given to:
  4. Images referenced in page/post content (in-body images)
  5. Featured images from live posts
  6. Product/service gallery images from EF
  7. Featured image migration for pages — 0 of 39 staging pages have featured images. At minimum, all key landing pages need featured images set for proper social sharing and search appearance.
  8. Featured image audit on live EF — 72 of 74 EF pages lack featured images. This is a pre-existing gap, not a migration regression, but should be addressed as part of the migration improvement scope.
  9. Run precise comparison scripts — Execute node D:/Website_Audit/scripts/media_audit.js to get exact filename match counts between live and staging. This will produce the definitive gap list for the media migration team.
