Section 2: CRITICAL Findings (Fix Before Migration)

CRITICAL-0: Fragmented Redirect Architecture (NEW)

Risk: The entire redirect strategy has no verified implementation path

The 8,526 migration redirect mappings from StudioHawk need to be implemented in a single, unified redirect engine. Currently:

  • EF uses 301 Redirects Pro 6.27 — a standalone plugin with its own database table. Existing rules are unknown (no REST API). These rules WILL interact with migration redirects.
  • BF has NO redirect plugin — only Easy HTTPS Redirection (SSL forcing, not URL redirects). BF's 1,655 backlinks and all page URLs have zero redirect infrastructure.
  • LUL uses Redirection 5.7.5 (John Godley plugin) — has REST API, confirmed 0 existing rules. Installed but empty.
  • MB uses Yoast SEO Premium 15.2.1 built-in redirect manager — no REST API for redirect rules. Existing rules unknown.
  • Staging has Permalink Manager Pro 2.5.1.1 AND Rank Math SEO PRO — both can handle redirects. Having two redirect engines creates chain/conflict risk. The rankmath/v1/updateRedirection endpoint exists, meaning Rank Math's redirect module IS enabled.

Verified via SSH (WP-CLI database queries on all 6 sites):

Critical finding: StudioHawk's 8,526 migration redirects have NOT been loaded anywhere. Staging has only 65 Rank Math rules (internal restructuring). The migration redirect map does not exist in any database on any site.

Questions that must be answered before migration:

  1. Where will the 8,526 redirects be loaded? (Rank Math on staging is the logical choice — it already has 65 rules)
  2. EF's 3 existing 301 Redirects Pro rules — will they conflict or be superseded after DNS cutover?
  3. LUL's 190 existing Rank Math rules — will they chain with migration redirects?
  4. Staging has both Permalink Manager Pro AND Rank Math — which takes priority? Disable one.

Fix:

  • Choose Rank Math on staging as the single redirect engine (already has the module active)
  • Load the 8,526 StudioHawk rules into Rank Math (import CSV feature available)
  • Cross-reference LUL's 190 existing rules against the migration map — collapse any chains
  • Disable Permalink Manager Pro's redirect function on staging to prevent conflicts
