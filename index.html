Executive Summary

  • Scope: 4-domain consolidation — Events Fantastic, Backdrops Fantastic, Light Up Letters, and Modular Backdrops — migrating into a single WooCommerce installation on WP Engine staging.
  • Method: 20 technical checks completed across all sites using full WordPress admin API access, direct SSH database queries across all 6 WP Engine environments, filename-by-filename media comparison, and full child theme code review.
  • Verdict: 46.5 / 100 — STOP. The migration is not ready to proceed. The staging site has critical infrastructure gaps that would cause immediate ranking and traffic loss.
  • Primary blocker: Redirect infrastructure is fragmented across 5 different plugins with 5 different configurations. 0 of 8,526 required migration redirects have been implemented on any environment. Every legacy URL will return a 404 on Day 1.
  • Second blocker: Filename-by-filename media comparison reveals 10,786 images missing from staging (only 17.3% match rate) — significantly worse than the earlier count-based estimate of ~10,459. LUL has just 2.5% match rate and MB just 3.5%.
  • New findings: ACF field audit uncovered 393 actionable database rows with old domain references (including 156 dual canonical conflicts on LUL). Child theme code review found 4 security vulnerabilities including XSS, ~190KB of globally loaded unused scripts, and NitroPack installed but not connected on any site.
  • Path to GO: 9 critical fixes and 13 high-priority fixes required. A re-audit after remediation is recommended before any DNS changes.

Critical Findings (9)

Critical C1. Redirect Architecture Fragmented Across 5 Plugins

Five different redirect mechanisms are active across the six WP Engine environments. There is no single source of truth for redirect rules, and plugins will conflict post-migration.

Database query results:
  • Events Fantastic: 301 Redirects Pro — 3 rules
  • Backdrops Fantastic: No redirect plugin — zero redirect capability
  • Light Up Letters: Rank Math — 190 rules
  • Modular Backdrops: No redirect plugin — zero redirect capability
  • Staging: Rank Math — 65 rules (internal restructuring only)
Recommendation Consolidate to a single redirect engine (Rank Math on staging). Load all 8,526 migration redirects into Rank Math, then disable all competing redirect plugins before DNS cutover.
Critical C2. 0 of 8,526 Migration Redirects Implemented

StudioHawk's redirect mappings exist in spreadsheets but have not been loaded into any WordPress database on any environment. The 65 Rank Math rules on staging are internal URL restructuring — none correspond to cross-domain migration redirects.

Impact: On Day 1 of migration, every legacy URL across all 4 domains (eventsfantastic.com.au, backdropsfantastic.com.au, lightupletters.com.au, modularbackdrops.com.au) will return 404. All indexed pages, all inbound links, and all bookmarked URLs will break.
Recommendation Import the full 8,526 redirect ruleset into Rank Math on staging. Validate a statistically significant sample (minimum 10% per domain) before any DNS change. This is the single highest-priority blocker.
Critical C3. Rental Terms Page in Staging Trash

Page ID 44248 ("Rental Terms") was moved to trash on 30 March 2026. This page is the redirect destination for both the Events Fantastic and Backdrops Fantastic rental terms pages.

Current state: Returns HTTP 404. Any redirect pointing to this URL will deliver users to a dead page, defeating the purpose of the redirect.
Recommendation Restore this page from trash immediately. Verify content matches the live Events Fantastic rental terms page. Assign to the correct permalink before redirect import.
Critical C4. 4,197 Hardcoded Old Domain URLs in Content

Post content, metadata, and serialized option values contain hardcoded references to legacy domain URLs. These will resolve incorrectly or break after DNS migration.

Breakdown by domain:
  • backdropsfantastic.com.au: 293 references
  • lightupletters.com.au: 3,765 references (3,752 are self-referencing image URLs on the LUL site)
  • modularbackdrops.com.au: 133 references
  • Staging site: 6 remaining references

1,619 unique URLs identified across all sites.

Recommendation Run search-replace across the staging database using the provided 1,619-URL find-replace map. Replace all legacy domain references with the consolidated eventsfantastic.com.au domain. Verify serialized data integrity after replacement.
Critical C5. Contact Form 7 Missing from Staging

Contact Form 7 (with Dynamic Text Extension, CFDB7 data storage, and ReCaptcha integration) is active on all 4 live production sites. None of these plugins are installed on the staging environment.

Impact: Every contact form across the consolidated site will be non-functional on Day 1. Enquiry forms are a primary lead generation mechanism for event hire businesses.
Recommendation Install Contact Form 7, CF7 Dynamic Text Extension, CFDB7, and CF7 ReCaptcha on staging. Import or recreate all form configurations. Test submission and data storage before go-live.
Critical C6. Wordfence Security Missing from Staging

Wordfence (v8.1.4) is active on all 4 live sites, providing WAF, brute-force protection, malware scanning, and login security. The staging site has no security plugin installed.

Impact: The consolidated site will launch with zero application-layer security. No firewall, no brute-force protection, no malware scanning on Day 1.
Recommendation Install and configure Wordfence on staging before go-live. Import firewall rules, configure login security, and enable real-time IP blocking.
Critical C7. AggregateRating Schema and 46 Reviews Not Migrated

The live Events Fantastic site has 46 customer testimonials and the "Reviews for Schema" plugin generating AggregateRating structured data that produces star ratings in Google search results. Neither the plugin nor the review data exists on staging. 0 of 1,311 WooCommerce products have any ratings.

Impact: Loss of star rating rich snippets in SERPs. These rich results significantly impact click-through rates for competitive event hire keywords.
Recommendation Import all 46 testimonials to the staging database. Install and configure the Reviews for Schema plugin (or equivalent Rank Math review schema). Verify AggregateRating markup renders correctly on product pages.
Critical C8. 393 ACF Field Rows with Old Domain References

Direct SSH database queries against wp_postmeta found 15,054 rows containing old domain references. After filtering noise, 393 are actionable and will cause content, SEO, or navigation issues post-migration.

Breakdown by type:
  • 156 dual canonical URLs on LUL: Both Yoast AND Rank Math canonical fields are set — a plugin conflict that will produce unpredictable canonical signals to search engines
  • 149 ACF content blocks on MB: Contain hardcoded @backdrops.com.au email addresses in customer-facing content
  • 29 slider fields on BF: Old image URLs referencing backdropsfantastic.com.au media files
  • 24 Product schema rows on LUL: Structured data referencing lightupletters.com.au domain
  • 6 navigation menu items: Linking directly to old domain URLs
Recommendation Priority 1: Resolve the 156 dual canonical conflicts on LUL by removing the Yoast canonical entries (Rank Math is the target SEO plugin). Priority 2: Run search-replace on the 149 email references and 29 image URLs. Priority 3: Update Product schema and navigation menu references.
Critical C9. XSS Vulnerability in Product Search Page

Child theme code review identified a cross-site scripting (XSS) vulnerability in the product search page. User input is rendered into HTML without proper sanitisation or escaping. Three additional AJAX handlers were found with no nonce verification, allowing potential CSRF attacks.

Security findings:
  • XSS in product search: Search query reflected in page output without escaping — allows arbitrary script injection
  • 3 AJAX handlers without nonce verification: Endpoints accept requests without CSRF token validation
  • Combined risk: An attacker could craft a malicious search URL and inject scripts into the page for any visitor who clicks it
Recommendation Immediately sanitise and escape all user input in the product search template using esc_html() and esc_attr(). Add wp_verify_nonce() checks to all AJAX handler functions. These are security vulnerabilities that must be fixed before the site goes live.

High Priority Findings

High H1. SEO Plugin Mismatch Across Sites

Events Fantastic, Backdrops Fantastic, and Modular Backdrops use Yoast SEO. Light Up Letters and the staging site use Rank Math. Metadata has been intentionally rewritten on staging via Rank Math — not imported from Yoast — meaning original meta titles and descriptions from 3 of 4 sites were not carried over.

Recommendation Verify that high-traffic pages from Yoast-powered sites have equivalent or improved meta titles and descriptions in Rank Math on staging. Conduct a page-by-page comparison for the top 50 organic landing pages per site.
High H2. 210 WooCommerce Products Missing from Staging

Of the products expected on staging, 210 are absent. 149 are restructured Light Up Letters products (consolidated into different categories), 42 are genuinely dropped products with no redirect mapping, and 27 are discontinued drapes.

Recommendation For the 42 genuinely dropped products: create redirect rules to the nearest equivalent category or product page. For the 149 restructured LUL products: verify each has a redirect from its legacy URL to the new consolidated product page.
High H3. 3,886 EF Images Missing Alt Text

40% of the Events Fantastic media library (3,886 of 9,725 images) has no alt text. This impacts accessibility compliance and image search visibility.

Recommendation Bulk-generate alt text for product and portfolio images using product names and categories. Prioritise images on the top 100 organic landing pages. This can be done post-migration but should be scheduled.
High H4. Media Library Gap — 10,786 Images Missing (17.3% Match Rate)

Filename-by-filename comparison across all 4 live sites reveals 13,049 unique image filenames. Only 2,263 matched on staging (17.3%). This is significantly worse than the earlier count-based estimate of ~10,459.

Match rates by site:
  • Events Fantastic: 21.9% match rate — best of the four, but still missing ~78% of images
  • Backdrops Fantastic: 19.0% match rate
  • Light Up Letters: 2.5% match rate — near-total image loss
  • Modular Backdrops: 3.5% match rate — near-total image loss

Dimension mismatch: Staging images are consistently 2x the dimensions of live images (e.g. 1240×620 on staging vs 620×310 on live). This suggests images were re-uploaded or regenerated rather than migrated, and original thumbnails were not preserved.

Recommendation Migrate the 10,786 missing images from live sites to staging. Regenerate thumbnails after migration to ensure dimension consistency. Prioritise LUL and MB sites where match rates are below 5%. Cross-reference against active product pages to determine which images are in use.
High H5. 1,158 Oversized Images on Staging

1,158 images in the staging media library exceed 500KB. Many are uncompressed originals that will impact page load times and Core Web Vitals.

Recommendation Run bulk image optimisation (ShortPixel, Imagify, or WP-CLI with ImageMagick). Target maximum 200KB per image with WebP conversion. Can be done post-migration.
High H6. WPBakery Shortcode Content on EF and LUL Pages

Several pages on Events Fantastic and Light Up Letters contain WPBakery Page Builder shortcodes. If the staging theme does not include WPBakery, these shortcodes will render as raw text in content.

Recommendation Identify all pages containing WPBakery shortcodes. Either install WPBakery on staging or convert affected content to Gutenberg blocks / plain HTML before migration.
High H7. Staging Settings Drift

The staging site has incorrect WordPress general settings: site title, tagline/description, admin email, and timezone do not match the intended production configuration for the consolidated Events Fantastic site.

Recommendation Update Settings > General on staging to match the intended production values before go-live. Verify timezone is set to Australia/Brisbane (AEST).
High H8. Modular Backdrops Timezone Empty

The Modular Backdrops WordPress installation has an empty timezone_string setting, defaulting to UTC. This affects scheduled posts, cache expiry, and WooCommerce order timestamps.

Recommendation Set timezone to Australia/Brisbane on the Modular Backdrops site. While this will not affect the consolidated site directly, it should be corrected for any data exports or order timestamp reconciliation during migration.

The child theme enqueues 24 JavaScript files and 11 CSS stylesheets on every page load, regardless of whether the page uses them. This adds approximately 190KB of wasted JavaScript to every page request.

Duplicate libraries found:
  • 2 carousel libraries: Both Slick and Owl Carousel loaded globally

  • form-checkout.php: Calls order review template twice, doubling the HTML output on checkout pages

Configuration NitroPack Not Connected & Hardcoded Dev URLs

NitroPack performance plugin is installed but has empty credentials on all sites — it is not providing any caching, CDN, or optimisation benefits. Additionally, 15+ hardcoded references to the development domain remain in the child theme code.

  • NitroPack: Installed on all sites but credentials fields are empty — zero optimisation active
  • Dev domain references: 15+ instances of eventsfantastic.thefive.com.au hardcoded in child theme templates and configuration files
Recommendation Obtain NitroPack credentials from the account holder and connect the plugin before go-live. Search-replace all instances of the development domain URL in the child theme files. Conditionally load scripts and styles only on pages that need them to reduce the ~190KB global JS waste.
